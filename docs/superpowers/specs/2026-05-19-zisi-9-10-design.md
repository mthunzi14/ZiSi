## Section 1: Signal Quality (7 → 9)
**File:** `sentiment_analyzer.py`

### 1A. FinBERT Cross-Validation (Ensemble Approach B)
After the primary provider (Gemini/Groq/Cerebras) returns batch results, run FinBERT
on the same articles locally. Compare labels:
- Agreement: confidence += 2, tag `finbert_validated: true`
- Disagreement: confidence -= 1, tag `sentiment_conflict: true`

FinBERT always runs in cross-validation mode (not as primary unless all APIs fail).
This produces calibrated, cross-validated confidence scores at zero extra API cost.

### 1B. Provider Confidence Normalization
All providers map to the same 1–10 scale:
- FinBERT: `raw_score * 10` (score is 0.0–1.0 from transformers)
- Groq/Gemini/Cerebras: as-is (already 1–10 from prompt)
- VADER: `abs(compound) * 10` (compound is -1 to +1)
- Keywords: as-is (already integer)

Normalization applied in `_normalize_provider_confidence()` called before `_apply_source_quality()`.

## Section 2: Architecture Bug Fix (8 → 9)
**File:** `updown_trader.py`

### 2A. Markov Recording for UpDown Trades
`record_updown_outcome()` already calls `update_coin_rolling_wr()`.
Add `tracker.record(coin, "UP" if won else "DOWN")` in the same function.
Markov EV gate requires `sample_size >= 10` — it will start blocking bad windows
after approximately 10 resolved UpDown trades (~2-3 hours of trading).

## Section 3: Adaptability (7 → 9)
**Files:** `main.py`, `updown_trader.py`

### 3A. Oracle Activation Threshold (main.py)
`_get_adaptive_threshold()` currently requires ≥5 trades per hour to activate.
Lower to ≥3. At 15 total trades spread over multiple hours, some hours may have 3+.

### 3B. Per-Duration WR Tracker (updown_trader.py)
Track win rate per window duration (5m, 10m, 15m, 60m) in `_duration_wr: dict`.
After ≥10 trades on a duration: if WR < 40%, skip that duration.
Reset on session restart (in-memory only, no persistence needed).

## Section 4: Execution Quality (7.5 → 9)
**File:** `updown_trader.py`

### 4A. Spread Gate: 4% → 2%
`_live_spread > 0.04` → `_live_spread > 0.02`
Prediction market normal spread is 1-2%. 4% is illiquid noise.

### 4B. Price Drift Gate: 6% → 3%
`abs(_live_mid - entry_price) > 0.06` → `abs(_live_mid - entry_price) > 0.03`
Entry prices older than 90 seconds drift. 3% drift means the market moved.

## Section 5: Risk Management (8 → 9)
**Files:** `main.py`, `updown_trader.py`

### 5A. Circuit Breaker: −$5 → −$8
`_CIRCUIT_BREAKER_THRESHOLD = -5.0` → `-8.0`
5% of $100 balance is too tight. 8% allows normal variance without false halts.

### 5B. UpDown Session Loss Limit: $6 → $10
`UPDOWN_SESSION_LOSS_LIMIT = 6.0` → `10.0`
Same reasoning: 6% of $100 is too conservative for a 3-4 trade session.

### 5C. Position Ladder Entry
When composite score ≥ 0.85 (high conviction), enter 60% of computed size immediately.
Store in `_ladder_pending: dict` with coin, direction, remaining 40% size, entry_ts.
In `check_updown_early_exits()`, check if 3+ minutes elapsed AND momentum still
aligned (RSI still directional) → place the remaining 40%.
If momentum reversed within 3 minutes → cancel ladder, log `LADDER_CANCEL`.

## Section 6: NR Scanner Expansion (Win Rate)
**File:** `near_resolution_scanner.py`

### 6A. Approaching-Certainty Tier
Add a new price tier: 0.65–0.75 (YES) or 0.25–0.35 (NO) with < 1h remaining.
These markets are "approaching certainty" — not guaranteed but directionally committed.
Confidence: 0.78. Position size: 3% of balance (smaller than core NR trades).

### 6B. Minimum Liquidity Filter
Add `$200` minimum liquidity check for NR candidates on Polymarket.
NR trades in thin markets have high slippage and stale prices.
